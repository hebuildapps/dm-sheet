EXP 01
Data Warehouse & ETL (Apache Hop + MySQL)
  • 📌 Q: Data warehouse vs operational DB? A: DW is subject-oriented, integrated, historical; OLTP is transactional, normalized.
  • 🔑 Surrogate key? Artificial PK (e.g., customer_key) independent of business ID, used for SCD & performance.
  • ⚙️ Add Sequence role: Generates surrogate keys in ETL pipelines.
  • 🔄 ETL significance: Extract → Transform (clean, standardize) → Load into star schema for analytics.
🧪 Implementer twist: Design a streaming ETL pipeline that processes IoT sensor logs every 5 mins — replace CSV with Kafka source, still using surrogate keys.
EXP 03
WEKA for ML / Data Mining Tasks
  • 📚 ML types in WEKA: Classification (J48, RF, NB), Regression, Clustering (k-Means), Association (Apriori), Ensemble.
  • 🎯 Algorithm selection: Based on problem type (classification), dataset size, interpretability vs accuracy, cross-validation.
  • 📊 Evaluation: Accuracy, F1-score, ROC curves to compare J48 vs Random Forest.
🧪 Implementer mindset: Export WEKA model as Java code, embed into a real-time medical triage dashboard — predict diabetes risk from live patient vitals.
EXP 04
Data Preprocessing (Cleaning, PCA, Normalization)
  • 📐 Normalization vs Standardization: Norm scales to [0,1]; Std centers mean=0, std=1 (z-score).
  • 📉 PCA: Dimensionality reduction by projecting variance onto principal components.
  • 🔍 IQR method: Outlier detection using Q1-1.5*IQR and Q3+1.5*IQR.
🧪 Implementer challenge: Write a Python function that automatically detects columns with outliers (IQR), caps them, and reapplies PCA for a fraud detection pipeline.
EXP 05
Apriori Algorithm (Association Rules)
  • 🔄 Why iterative: Generates (k)-itemsets from (k-1) frequent itemsets until no candidates.
  • 📈 Low min support effect: Huge itemsets, computationally expensive, many trivial rules.
  • 🛒 Market Basket Analysis: Bread → Butter (lift>1) helps cross-selling.
🧪 Implementer twist: Build a real-time recommendation engine for e‑commerce using Apriori on streaming transaction data (Spark MLlib) — refresh rules hourly.
EXP 06
Text Mining + Naïve Bayes (IoT Alerts)
  • 🧠 Naïve Bayes suitability: High-dim sparse text data, efficient, probabilistic, works with small datasets.
  • 📝 TF-IDF purpose: Weighs term importance — downweights common words, boosts unique discriminators.
  • 🚫 No stopword removal: Noise, overfitting, slower training.
  • ⚡ Real-time IoT deployment: Edge gateway with pre-trained model, classify alerts (fire, fault) and trigger immediate actions.
🧪 Implementer's mindset: Deploy TF-IDF + Naïve Bayes as a REST API using FastAPI, subscribe to MQTT topics from smart factories, auto-flag critical anomalies.
